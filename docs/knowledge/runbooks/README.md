# Runbooks

Procedimientos de respuesta a incidentes y operación de plataforma: caída de servicio, fuga de datos sospechada, rollback de despliegue, recuperación de backups, incidentes de agentes IA o n8n.

Cada runbook define: trigger, severidad, responsables, pasos de mitigación, criterios de resolución y post-mortem requerido.
